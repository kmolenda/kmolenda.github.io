Arytmetyka modularna, algebra reszt

Intuicja

Arytmetyka modularna:
w matematyce, system liczb całkowitych, w którym liczby zawijają się po osiągnięciu pewnej wartości określonej terminem modulo (skracanie mod).

Przykłady:

  • obliczanie godzin na zegarze 24-godzinnym (np. ile godzin upłynęło od godz. 16 po południu do godziny 2 w nocy),
  • obliczenia binarne w rejestrze 8-bitowym (z uwzględnieniem overflow).
Definicja:
Niech a,b∈Z,m≠0a, b \in \mathbb{Z},m \neq 0a,b∈Z,m=0. Mówimy, że aaa przystaje do bbb modulo mmm i zapisujemy a≡mba \equiv_m ba≡m​b wtedy i tylko wtedy, gdy a−ba - ba−b jest wielokrotnością mmm (lub mówiąc inaczej, aaa i bbb mają tę samą resztę z dzielenia całkowitego przez mmm).
Relację przystawania nazywamy relacją kongruencji.

Fakty:

  • Relacja przystawania modulo mmm (≡m\equiv_m≡m​) jest relacją równoważności w zbiorze Z\mathbb{Z}Z. Dlatego czasami nazywana jest równością modulo mmm.

    Dla dowolnych aaa, bbb, ccc oraz m>0m>0m>0 zachodzi:

    • a≡maa \equiv_m aa≡m​a,
    • a≡mba \equiv_m ba≡m​b wtedy i tylko wtedy, gdy b≡mab \equiv_m ab≡m​a,
    • jeśli a≡mba \equiv_m ba≡m​b oraz b≡mcb \equiv_m cb≡m​c, to a≡mca \equiv_m ca≡m​c.

  • Oznaczenie: klasa równoważności [a]m[a]_m[a]m​ względem relacji przystawania modulo mmm.

  • Zbiór ilorazowy Z/≡m\mathbb{Z} / \equiv_mZ/≡m​ - zbiór reszt dzielenia przez mmm ({0,1,…,m−1}\{0, 1, \ldots, m-1\}{0,1,…,m−1}).

  • Relacja ≡m\equiv_m≡m​ jest zgodna z działaniami arytmetycznymi dodawania, odejmowania i mnożenia. Wprowadza się arytmetykę w zbiorze ilorazowym:

    • [a]m+[b]m=[a+b]m[a]_m + [b]_m = [a+b]_m[a]m​+[b]m​=[a+b]m​

Rozszerzony algorytm Euklidesa (NWDxy)

Podstawowa wersja algorytmu Euklidesa pozwala na wyznaczenie największego wspólnego dzielnika dla dwóch liczb naturalnych. Rozszerzona wersja rozwiązuje równanie diofantyczne:

NWD(a, b)=a⋅x+b⋅yNWD(a,\ b)=a\cdot x+b\cdot yNWD(a, b)=a⋅x+b⋅y

Dla zadanych aaa oraz bbb naturalnych wyznaczane są liczby całkowite xxx oraz yyy spełniające powyższe równanie.

// wersja rekurencyjna
public static int NWDxy(int a, int b, out int x, out int y)
{
    System.Diagnostics.Debug.Assert( a >= 0 && b > 0 );
    x = 0; y = 1;

    // przypadek podstawowy
    if (a == 0) return b;

    // zapamiętujemy wyniki wywołań rekrencyjnych
    int x1 = 1, y1 = 1;
    int nwd = NWDxy(b % a, a, out x1, out y1);

    // aktualizujemy x oraz y wykorzystując wyniki rekurencji
    x = y1 - (b / a) * x1;
    y = x1;

    return nwd;
}

// przykłady
// 35*x + 15*y = nwd(35,15)
int x, y;
int a = 35, b = 15;
int wynik = NWDxy(a, b, out x, out y);
Console.WriteLine($"{a}*x + {b}*y = NWD({a},{b})");
Console.WriteLine($"{a}*({x}) + {b}*({y}) = {wynik}");

Potęgowanie modularne

Potęgowanie modularne wyraża się wzorem: abmod  na^b \mod nabmodn

Algorytm:

  • Jeśli a≥na \geq na≥n, to wyznaczamy a′a'a′ jako resztę z dzielenia aaa przez nnn, a dopiero potem (a′)bmod  n(a')^{b} \mod n(a′)bmodn, jako że (a′)bmod  n=abmod  n(a')^{b}\mod n = a^{b} \mod n(a′)bmodn=abmodn.
  • Należy zatem rozważać sytuację 0≤a<n0 \leq a < n0≤a<n.
  • W przeciwieństwie do zwykłego potęgowania wynik nigdy nie przekracza nnn, czyli nie rośnie wykładniczo w stosunku do aaa. Pozwala to żywić nadzieję na szybsze algorytmy potęgujące.

Algorytm naiwny:

  • Wymnóż aaa przez siebie bbb-krotnie, po każdym mnożeniu weź resztę modulo nnn.
  • Branie reszty po każdym mnożeniu, utrzymuje tymczasowy iloczyn w zbiorze {0,…,n−1}\left\{{0,\ldots ,n-1}\right\}{0,…,n−1}.

System kryptograficzny RSA

Funkcja kodująca fff to permutacja

f:J⟶Jf:{\mathcal {J}}\longrightarrow {\mathcal {J}}f:J⟶J

gdzie J\mathcal {J}J to zbiór wszystkich, możliwych jednostek tekstu jawnego i szyfrogramu (w skrócie jednostki wiadomości).

Funkcja dekodująca jest permutacją odwrotną do funkcji kodującej. Oznaczamy ją przez f−1f^{-1}f−1, gdzie fff jest odpowiednią funkcją kodującą.

Funkcja kodująca/dekodująca zależy zazwyczaj od parametrów, zwanych kluczami. Przyjmuje się, iż funkcja kodująca (jej wzór, bądź algorytm ją realizujący) jest ogólnie dostępna. Zachowane w tajemnicy są jedynie klucze.

Kryptosystem to para (J,f)({\mathcal {J}},f)(J,f), czyli zbiór jednostek wiadomości J\mathcal {J}J i funkcja kodująca fff.

Kryptoanaliza to dział kryptografii badający możliwość dekodowania wiadomości bez znajomości kluczy użytego kryptosystemu. W ogólności badane są tu też możliwości zamiany wiadomości, fałszowania podpisów, itp.

Klucz publiczny

  • Klucz kodowania - KKK_{K}KK​ to parametr(y) funkcji kodującej. Znając je można wysyłać zakodowane wiadomości.

  • Klucz dekodowania - KDK_{D}KD​ to paramert(y) funkcji dekodującej. Znając je można dekodować wiadomości.

Przez wieki uważano, że w dowolnym kryptosystemie znajomość klucza kodujacego KKK_{K}KK​ pozwala na szybkie poznanie klucza dekodującego KDK_{D}KD​, czyli że każdy kto umie kodować wiadomości może je też dekodować. Tak jest np. w kryptosystemie Cezara (opartym na dodawaniu modulo). Jeśli Cezar używał tego samego kryptosystemu do kontaktu ze wszystkimi swoimi generałami, wtedy mogło dochodzić do sytuacji niepożądanych, np. gdy generał A wysłał Cezarowi zaszyfrowaną wiadomość, generał B przechwytując kuriera po drodze mógł ją odczytać.

Kryptosystem z kluczem publicznym to system (J,f,KK,KD)({\mathcal {J}},f,K_{K},K_{D})(J,f,KK​,KD​), gdzie KKK_{K}KK​ i KDK_{D}KD​ są odpowiednio kluczem kodującym i dekodującym, w którym:

  • KKK_{K}KK​ jest powszechnie dostępny;
  • KDK_{D}KD​ jest zachowywany w tajemnicy,
  • znajomość KKK_{K}KK​ pozwala na szybkie kodowanie jednostki tekstu, czyli obliczanie f(P)f(P)f(P) dla P∈JP\in {\mathcal {J}}P∈J,
  • znajomość KKK_{K}KK​ nie pozwala (bez znajomości KDK_{D}KD​) na szybkie dekodowanie jednostki szyfrogramu, czyli obliczanie f−1(C)f^{-1}(C)f−1(C) dla C∈JC\in {\mathcal {J}}C∈J,
  • znajomość KDK_{D}KD​ daje możliwość szybkiego zdekodowania jednostki szyfrogramu.

Szukamy więc funkcji fff, którą w jedną stronę liczy się relatywnie szybko na podstawie znajomości KKK_{K}KK​. Natomiast jej odwrotna f−1f^{-1}f−1 jest bardzo trudna do policzenia bez dodatkowej informacji KDK_{D}KD​.

Przykład:

  1. Obliczenie iloczynu dwóch bardzo długich liczb całkowitych jest problemem wielomianowym (relatywnie proste).
  2. Rozkład wyniku z (1) na czynniki pierwsze jest problemem wykładniczym (bardzo trudnym obliczeniowo).
  3. Jeśli znalibyśmy jeden z czynników rozkładu, to ten drugi czynnik obliczymy szybko, wykonując dzielenie całkowite.
  4. Wszystkie powyższe założenia dotyczą tradycyjnej architektury komputera (von Nuemann'a), w przypadku komputera kwantowego wszystko jest proste i szybko osiągalne.

Na bazie powyższych spostrzeżeń oparta jest idea kryptosystemu RSA (Rivest, Shamir i Adelman, 1977).

Opis działania RSA

  1. Wybierz losowo dwie bardzo duże różne liczby pierwsze ppp i qqq.

    • To jak duże mają być to liczby zależy od aktualnej mocy obliczeniowej komputerów. Oczywiście, im większe liczby tym trudniej złamać kod (KDK_{D}KD​) ale jednocześnie, im większe liczby, tym dłużej trwa proces kodowania i dekodowania wiadomości.
    • To, że liczba ma być wybrana losowo oznacza, że jest uzyskana z pomocą generatora liczb pseudo-losowych. Ale co to oznacza wybrać (pseudo)losowo liczbę pierwszą?
      • Wylosuj odpowiednio dużą liczbę mmm. Jeśli jest ona parzysta, to podstaw m←m+1m \leftarrow m + 1m←m+1.
      • Zaaplikuj testy pierwszości do mmm. Jeśli okaże się złożona, podstaw m←m+2m \leftarrow m + 2m←m+2 i tak, aż do znalezienia liczby pierwszej. Z Twierdzenia o Liczbach Pierwszych wiemy, że częstotliwość występowania liczb pierwszych w pobliżu liczby mmm wynosi 1ln⁡m\frac {1}{\ln {m}}lnm1​, a więc możemy oczekiwać że po O(lg⁡m)O(\lg {m})O(lgm) próbach test pierwszości zwróci wynik pozytywny.

  2. Niech n=p⋅qn=p\cdot qn=p⋅q. Wtedy φ(n)=φ(p)⋅φ(q)=(p−1)⋅(q−1)\varphi (n)=\varphi (p)\cdot \varphi (q)=(p-1)\cdot (q-1)φ(n)=φ(p)⋅φ(q)=(p−1)⋅(q−1).

    • Wybierz losowo e∈{1,2,3,…,φ(n)−1}e\in \left\{{1,2,3,\ldots ,\varphi (n)-1}\right\}e∈{1,2,3,…,φ(n)−1} względnie pierwsze z φ(n)\varphi (n)φ(n).
    • Niech d∈{1,…,φ(n)−1}d\in {\left\{{1,\ldots ,\varphi (n)-1}\right\}}d∈{1,…,φ(n)−1} takie, że e⋅d ≡φ(n)1e\cdot d\ \equiv _{\varphi (n)}1e⋅d ≡φ(n)​1.
      • Podobnie jak wcześniej szukaliśmy losowo liczb pierwszych, teraz możemy szukać liczby eee. Zamiast aplikować test pierwszości sprawdzamy algorytmem Euklidesa względną pierwszość z φ(n)\varphi (n)φ(n). Kiedy uda nam się trafić (statystycznie po O(log⁡φ(n))O(\log {\varphi (n)})O(logφ(n)) krokach), rozszerzony algorytm Euklidesa wyznaczy nam ddd.

  3. Opublikuj klucz publiczny: KK=(n,e)K_{K}=(n,e)KK​=(n,e) i zachowaj do własnego użytku klucz dekodujący KD=(n,d)K_{D}=(n,d)KD​=(n,d).

  4. Zdefiniuj:

    • zbiór jednostek wiadomości jako Zn={0,1,…,n−1}\mathbb {Z}_{n}={\left\{{0,1,\ldots ,n-1}\right\}}Zn​={0,1,…,n−1} - na przykład każdemu znakowi przypisujesz wartość ze zbioru Zn\mathbb {Z}_{n}Zn​,

    • Problem RSA: znajdź eee-ty pierwiastek liczby C∈ZnC\in \mathbb {Z}_{n}C∈Zn​ modulo nnn, a więc posiadając e,n oraz Ce, n \ \text{oraz} \ Ce,n oraz C, znajdź takie PPP, że Pe ≡nCP^{e} \ \equiv_{n} CPe ≡n​C.
