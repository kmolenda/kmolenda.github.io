Kontrola przepływu sterowania (flow of control)

Krzysztof Molenda 2019-03-20 C#

Wszystkie języki programowania imperatywnego wprowadzają instrukcje sterujące biegiem programu. W języku C# są nimi:

  • Instrukcje warunkowego wyboru (selection statements)

    • if, if-else, if-elseif
    • switch-case

  • Wyrażenia warunkowego wyboru (selection expressions)

    • Operator warunkowego wyboru: _?_:_
    • Operator null-coalescing _??_

  • Instrukcje iteracyjne (pętle) (iteration statements)

    • while
    • do-while
    • for
    • foreach

  • Instrukcje przekazywania sterowania (transfer statements)

    • break, continue, etykiety
    • return
    • propagacja i obsługa wyjątków: try-catch-finally, throw
    • asercje: Debug.Assert

Instrukcje i wyrażenia warunkowego wyboru (selection statements)

  • instrukcje: if, if-else, if-elseif-else, switch
  • wyrażenia warunkowego wyboru: _?_:_, _??_

Prosty wybór: if

Składnia if

if( war_log )
  instr_lub_blok_instr

gdzie:

  • war_log - wyrażenie zwracające wartość logiczną (true lub false),
  • instr_lub_blok_instr – instrukcja pusta (;), pojedyncza instrukcja lub blok instrukcji (w klamrach {...})

Semantyka if

  1. Ustalenie wartości warunku logicznego
  2. Jeśli warunek logiczny przyjmuje wartość true, wykonanie instrukcji lub bloku instrukcji, a następnie przejście do kolejnej instrukcji.
  3. Jeśli warunek logiczny przyjmuje wartość false – brak jakiejkolwiek akcji i przejście do kolejnej instrukcji.

Inaczej niż w językach C i C++ wyrażenie testujące musi zwracać wartość typu bool. W C# liczb (np. 0 czy 1) nie można interpretować jako wartości bool.

Przykłady (if, cz. 1)

// pojedyncza instrukcja, nawiasy klamrowe nie potrzebne
if( temperatura > krytyczna )
    wywolajAlarm(); 

// wiele instrukcji - konieczny blok (nawiasy klamrowe)
if( t == 0 )
{
    x = x – 2;
    t++;
}

// błąd: w if( ... ) musi być wyrażenie logiczne, nie int
int x = 1;
if( x )
{
    ...
}

Przykłady (if, cz. 2)

Przykład 1a

if( temperatura > krytyczna );
    wywolajAlarm();

Komentarz – błąd semantyczny:

  • Wprowadzenie średnika (;) w pierwszym wierszu oznacza umieszczenie instrukcji pustej.
  • Metoda wywolajAlarm() wykona się zawsze, mimo sugestywnego wcięcia!

Kod równoważny, czytelniejszy:

if( temperatura > krytyczna )
    ; //instrukcja pusta - nic nie rób
wywolajAlarm();

Przykład 1b

if( temperatura > krytyczna )
    wywolajAlarm();

Komentarz:

  • Średnik (;) w wierszu wywolajAlarm() jest konieczny i oznacza – w tym przypadku – zakończenie instrukcji if( ).
  • Metoda wywolajAlarm() wykona się tylko wtedy, gdy spełniony będzie warunek if( ).
Przykład 2a
if( t = 0 )
{
    x = x – 2;
    t++;
}

Komentarz – błąd syntaktyczny:

  • Częstą pomyłką jest użycie operatora podstawienia (=) zamiast operatora równości (==).
  • Kompilator zgłosi błąd: oczekiwany typ logiczny!

Przykład 2b

if( t == 0 )
{
    x = x – 2;
    t++;
};

Komentarz:

  • Nie ma potrzeby umieszczania średnika po bloku zamykającym instrukcję if.
  • Tu nie zostanie zgłoszony błąd, ale średnikiem wprowadziliśmy, niepotrzebnie, instrukcję pustą (po wykonaniu if „nic nie rób”, a potem idź dalej).

Wybór złożony - if-else

Składnia if-else

if( war_log )
    instr_lub_blok_instr1
else
    instr_lub_blok_instr2

gdzie:

  • war_log - wyrażenie zwracające wartość logiczną (true lub false),
  • instr_lub_blok_instr1, instr_lub_blok_instr2 – instrukcja pusta (;), pojedyncza instrukcja lub blok instrukcji (w klamrach {...})

Semantyka if-else

  1. Ustalenie wartości warunku logicznego.
  2. Jeśli wynik true – wykonanie instrukcji lub bloku instrukcji 1, a następnie przejście do kolejnej instrukcji.
  3. Jeśli wynik false – wykonanie instrukcji lub bloku instrukcji 2, a następnie przejście do kolejnej instrukcji.

Przykład if-else

if( temperatura > krytyczna )
{
   wylaczReaktor();
   wywolajAlarm();
   ewakuacja();
}
else
   dostarczajEnergie();

Przykłady if-else

Przykład 1

if( temperatura > gornyLimit )
{
    if( niebezpieczenstwo ) wywolajAlarm();
}
else
    wlaczReaktor();

Komentarz:

  • Alarm zostanie wywołany, gdy temperatura przekroczy górny limit i niebezpieczenstwo == true.
  • Reaktor zostanie włączony, gdy temperatura nie przekroczy górnego limitu, bez względu na stan zmiennej niebezpieczenstwo.
  • W tego typu wnioskowaniach odwrotnych stosujemy prawa de Morgana z logiki.

Przykład 2

Usunięto nawiasy klamrowe bloku z przykładu 1.

if( temperatura > gornyLimit )
    if( niebezpieczenstwo ) wywolajAlarm();
else
    wlaczReaktor();

Komentarz:

  • Alarm zostanie wywołany, gdy temperatura przekroczy górny limit i niebezpieczenstwo == true
  • Włączenie reaktora będzie miało miejsce, gdy temperatura nie przekroczy górnego limitu i nie będzie niebezpieczeństwa (niebezpieczenstwo == false).

Ważne: Wiązanie klauzuli else z if

Klauzula else zawsze odnosi się do najbliższego if, który nie jest jeszcze powiązany z inną klauzulą else.
Zastosowanie notacji bloku {...} rozwiązuje problem niejednoznaczności.

  • if z linii (1) jest powiązane z else w linii (5)
if( ... )
{
    if(...)
}
else ...
  • if z linii (2) jest powiązany z else w linii (3)
if( ... )
    if(...)
else ...

Przykład 3

if( temperatura > gornyLimit && niebezpieczenstwo ) 
    wywolajAlarm();
else
    wlaczReaktor();

Komentarz:

  • Alarm zostanie wywołany, gdy temperatura przekroczy górny limit i niebezpieczenstwo == true.
  • Włączenie reaktora będzie miało miejsce, gdy temperatura nie przekroczy górnego limitu lub nie będzie niebezpieczeństwa (niebezpieczenstwo == false).

Przykład 4

if( temperatura > gornyLimit || niebezpieczenstwo )
    wywolajAlarm();
else
    wlaczReaktor();

Komentarz:

  • Alarm zostanie wywołany, gdy temperatura przekroczy górny limit lub niebezpieczenstwo == true.
  • Włączenie reaktora będzie miało miejsce, gdy temperatura nie przekroczy górnego limitu i nie będzie niebezpieczeństwa (niebezpieczenstwo == false).

Kaskadowe użycie if-else

if ( temperatura >= MAX )
{
    zaswiecKontrolke( CZERWONA );
    wylaczGrzejnik();
}
else if ( temperatura < MIN ) 
{
    zaswiecKontrolke( ZOLTA );
    wlaczGrzejnik();
}
else if ( temperatura == (MAX-MIN)/2 ) 
    zaswiecKontrolke( ZIELONA );
else
    zaswiecKontrolke( ZIELONA, PULSUJACA );

  • Jeśli którykolwiek z bloków if() wykona się (tzn. jego warunek logiczny ma wartość true, co w konsekwencji prowadzi do wykonania instrukcji tego bloku), następujące po nim bloki if-else są omijane.

  • nnn-ty blok if-else jest wykonywany tylko wtedy, gdy wszystkie n−1n-1n−1 warunków if-else go poprzedzających wartościuje się na false.

  • Jeśli żaden z warunków nie wartościuje się na true, wykonywany jest blok związany z ostatnią klauzulą else. Jeśli nie ma ostatniej klauzuli else, nic się nie wykonuje.

Operator warunkowego wyboru _?_:_

Trójargumentowy (ternarny) operator o składni:

war_log ? wyr1 : wyr2

gdzie:

  • war_log – wyrażenie zwracające wartość logiczną (true lub false),
  • wyr1, wyr2 – wyrażenia zwracające wartości porównywalnych typów

Semantyka _?_:_

  1. Operator zwraca wartość (może być użyty w kontekście operacji przypisania),
  2. Jeżeli war_log == true, zwraca wartość wyr1
  3. Jeżeli war_log == false, zwraca wartość wyr2

Przykłady _?_:_

Przykład 1

int x, y, max;
// ...
max = (x < y) ? y : x;

Komentarz: zwraca maksimum dwóch liczb.

Zapis równoważny z użyciem instrukcji if:

if( x < y )
    max = y;
else
    max = x;

Przykład 2

bool awaria;
// ...
Console.WriteLine( awaria ? "Alarm" : "Wszystko jest OK" );

Przykład 3

bool rokPrzestepny;
// ...
int liczbaDniLutego = rokPrzestepny ? 29 : 28;

Zagnieżdżanie wyrażeń warunkowego wyboru

Wyrażenia warunkowe mogą być zagnieżdżane, a operator wiązany jest od prawej do lewej strony.

Wyrażenie:

(a?b?c?d:e:f:g)

jest wartościowane następująco:

(a?(b?(c?d:e):f):g)

Operator wyrażenia warunkowego jest równoważny instrukcji if-else, ale zwraca on wartość, którą należy przechwycić/wysłać/wykorzystać.

Instrukcja if-else nie zwraca wartości.

❗️ Operator wyrażenia warunkowego powinien być stosowany dla zmniejszenia ilości kodu, ale z uwzględnieniem utrzymania jego czytelności !

Przykład - maksimum 3 liczb

W zapisie tradycyjnym:

int max3(int a, int b, int c) 
{
  int max;
  if( a < b )
    if( b < c ) max = c;
    else max = b;
  else
    if( c > a ) max = c;
    else max = a;
  return max;
}

Z użyciem operatora warunkowego wyboru:

int max3( int a, int b, int c )
{
  return (a < b)? (b < c)? c : b : (c > a)? c : a;
}

lub jeszcze szybciej, z użyciem notacji lambda:

int max3( int a, int b, int c ) => (a < b)? (b < c)? c : b : (c > a)? c : a;

Więcej informacji: ?: Operator (C# Reference)

  1. Napisz i przetestuj funkcję max4 wyznaczającą maksimum 4 liczb - wariantach tradycyjnym oraz z użyciem operatora warunkowego wyboru.

  2. Wykonaj wcześniejsze zadanie dla 5 liczb.

Null-coalescing operator ??

Operator upraszczający zapis warunkowego wyrażenia, jeśli wartość wyrażenia jest null (czyli jest nieokreślona).

The ?? operator is called the null-coalescing operator and is used to define a default value for nullable value types or reference types. It returns the left-hand operand if the operand is not null; otherwise, it returns the right operand. The C# Null Coalescing Operator (??) is a binary operator that simplifies checking for null values. It is used to assign a default value to a variable when the value is null.

Przykład 1 - dla typów referencyjnych

Poniższe fragmenty kodu są równoważne:

string pageTitle = suppliedTitle ?? "Default Title";

string pageTitle = (suppliedTitle != null) ? suppliedTitle : "Default Title";

string pageTitle = !(suppliedTitle is null) ? suppliedTitle : "Default Title";

string pageTitle;
//if( !(suppliedTitle is null) )
if( suppliedTitle != null )
{
    pageTitle = suppliedTitle;
}
else
{
    pageTitle = "Default Title";
}

Przykład 2 - dla typów nullable

int? x = null;

// Przypisz zmiennej y wartość x, jeśli x nie jest równe null;
/// w przeciwnym przypadku, gdy x == null, podstaw do y wartość -1.
int y = x ?? -1;

Prawostronna łączność operatora ??

Operator ?? może być wielokrotnie użyty w tym samym wyrażeniu. Operator jest prawostronnie łączny - ewaluacja wyrażenia następuje od prawej do lewej.

Na przykład wyrażenie postaci a ?? b ?? c jest ewaluowane jako a ?? (b ?? c).

Wynika z tego, że wyrażenie postaci E1 ?? E2 ?? ... ?? En zwraca wartość pierwszego operanda który nie jest null albo wartość null jeśli wszystkie są null.

Więcej informacji: ?? operator (C# Reference)

Wielokrotny wybór (switch-case)

Koncepcyjnie – instrukcja switch-case może być użyta, w celu wybrania jednej spośród wielu akcji, na podstawie wartości testowanego wyrażenia.

Składnia switch-case

switch (wyrażenie)
{
    case etykieta1: instr_lub_blok1
    case etykieta2:
    case etykieta2: instr_lub_blok2
    ...
    case etykietaN: instr_lub_blokn
    default: instr_lub_blok
} //koniec switch

Do wersji C# 6 włącznie:

  • Typ testowanego wyrażenia: char, byte, short, int, string, enum.
  • ❗️ Typ testowanego wyrażenia nie może być typu zmiennoprzecinkowego (float, double) !!!

Od wersji C# 7 wprowadzono istotne rozszerzenia funkcjonalne tej instrukcji. W switch() można umieścić dowolne wyrażenie nie zwracające null. Zagadnienie tzw. pattern expressions (w C# 7) jest poza zakresem niniejszego wykładu.

Semantyka switch-case

  1. Określana jest wartość wyrażenia.
  2. Wartość tego wyrażenia jest porównywana z etykietami case.
  3. Kontrola jest przekazywana do instrukcji lub bloku instrukcji odpowiadającemu pierwszej etykiecie, której wartość jest równa wartości wyrażenia.
    Po wykonaniu tej instrukcji lub listy instrukcji sterowanie przekazane jest jawnie inną instrukcją, np. goto case, return, throw, lub najczęściej break). ❗️ Uwaga: inaczej niż w C++
  4. Jeśli nie ma etykiety równej wartości wyrażenia switch, wykonywana jest instrukcja lub blok instrukcji związany etykietą default. Jeżeli brak jest etykiety default (jest ona opcjonalna), cała instrukcja switch jest pomijana.

... ... ...

  • Wszystkie etykiety (włącznie z etykietą default) są opcjonalne i mogą być umieszczane w ciele instrukcji switch w dowolnym porządku. Porządek ten powinien odzwierciedlać kolejność wyszukiwania pasującej etykiety.
  • W ciele instrukcji switch może wystąpić co najwyżej jedna etykieta default.
  • Etykiety case są stałymi wyrażeniami, których wartości muszą być unikalne (w sensie: nie mogą się powtarzać). Wartości etykiet muszą być „przypisywalne” do typu wyrażenia switch. W szczególności, nie mogą przekraczać zakresu tego typu.
  • Aby przerwać ciąg wykonań kolejnych bloków (po odnalezieniu etykiety pasującej), stosuje się instrukcję break przerywającą działanie instrukcji switch. Powinna ona być ostatnią w bloku.

Przykłady

Przykład 1

int caseSwitch = 3;
switch (caseSwitch)
{
    case 1: Console.WriteLine("Case 1"); break;
    case 2: case 3: Console.WriteLine("Case 2"); break;
    default: Console.WriteLine("Default case"); break;
}

Przykład 2

int switchExpression = 3;
switch (switchExpression)
{
    // A switch section can have more than one case label.
    case 0:
    case 1:
        Console.WriteLine("Case 0 or 1");
        // Most switch sections contain a jump statement, such as
        // a break, goto, or return. The end of the statement list
        // must be unreachable.
        break;
    case 2:
        Console.WriteLine("Case 2");
        break;
        // The following line causes a warning.
        Console.WriteLine("Unreachable code");
        // 7 - 4 in the following line evaluates to 3.
    case 7 - 4: // 7-4 == 3
        Console.WriteLine("Case 3");
        break;
    // If the value of switchExpression is not 0, 1, 2, or 3, the
    // default case is executed.
    default:
        Console.WriteLine("Default case (optional)");
        // You cannot "fall through" any switch section, including
        // the last one.
        break;
}

Przykład 3

Console.WriteLine("Coffee sizes: 1=small 2=medium 3=large"); 
Console.Write("Please enter your selection: ");
//string str= Console.ReadLine();
string str = "2";
Console.WriteLine(str);
int cost = 0;
// Notice the goto statements in cases 2 and 3. The base cost of 25 
// cents is added to the additional cost for the medium and large sizes. 
switch (str)
{ 
    case "1": case "small":
        cost += 25;
        break;
    case "2": case "medium":
        cost += 25;
        goto case "1"; 
    case "3": case "large":
        cost += 100;
        goto case "1";
    default:
        Console.WriteLine("Invalid selection. Please select 1, 2, or 3.");
        break;
}
if (cost != 0)
    Console.WriteLine("Please insert {0} cents.", cost);

Console.WriteLine("Thank you for your business.");
