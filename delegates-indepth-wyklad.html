Delegaty są obiektami

Delegat jest typem użytkownika, podobnie jak klasa. Jednakże, o ile klasa reprezentuje zbiór danych i metod, o tyle delegat przechowuje jedną (lub wiele - o tym później) metod (plus zbiór predefiniowanych operacji).

Aby użyć delegatów w kodzie, należy wykonać następujące kroki:

  1. Zadeklarować typ delegata. Przypomina to deklarację metody w interfejsie (tzn. sam nagłówek, bez ciała).

  2. Zadeklarować zmienną typu delegatowego.

  3. Utworzyć obiekt typu delegatowego (tak - instancja typu delegatowego jest obiektem, bo w C# w pewnym sensie wszystko jest klasą/obiektem). Nowo utworzony delegat (w rozumieniu instancji typu delegatowego) zawiera referencję do metody o dokładnie takiej sygnaturze i zwracanym typie jak delegat (w sensie typu) zdefiniowany w kroku pierwszym.

  4. W innym fragmencie kodu można skorzystać z tak utworzonego delegata (rozumianego jako obiekt), wywołując go (ang. call, invoke) tak samo, jak metodę zdefiniowaną w klasie.

W porównaniu do pracy z klasami mamy bardzo duże podobieństwo.

⚠️ Na podanym rysunku celowo wykorzystano metodę tworzenia obiektów typu delegate stosowaną w C# 1 (z operatorem new) po to, aby pokazać podobieństwo między klasami a delegatami. W kolejnych wersjach zastosowano uproszczenia (bez użycia operatora new czy jako lambda wyrażenie).

Typ delegate jest typem referencyjnym. Definiowany jest w specjalnej abstrakcyjnej klasie Delegate (a później MulticastDelegate). Specjalność tej klasy polega na tym, że:

  • jest abstrakcyjna, ale można ją "implementować" wyłącznie za pomocą słowa kluczowego delegate opisując typ delegatowy (formalnie: deklaracja typu delegatowego jest implementacją klasy z MulticastDelegate),
  • dostarcza mechanizmów tworzenia i operowania na obiektach typu delegatowego -- jest klasą bazową dla wszystkich typów delegatowych,
  • nie można z niej dziedziczyć (może to robić tylko środowisko .Net i kompilator)
  • klasa Delegate nie jest rozpatrywana jako typ delegate.

Mimo, iż klasa Delegata wyposażona jest w konstruktory, nie można ich jawnie użyć. Można natomiast użyć jednej z przeciążonych metod Createdelegate.

Wnętrze delegata

Możesz postrzegać delegata jako obiekt, który zawiera uporządkowaną listę metod o tej samej sygnaturze i takim samym typie zwracanym.

  • lista metod nazywana jest Invocation List
  • w tej liście przechowywane są referencje do metod zdefiniowanych w dowolnej klasie lub strukturze (oczywiście zgodnych z typem delegatowym)
  • metody mogą być metodami instancji jak również metodami statycznymi czy anonimowymi
  • gdy delegat zostaje wywołany (ang. call, invoked), wszystkie metody z listy zostaną wywołane (uruchomione) w kolejności zapisania w liście inwokacyjnej - o ile są )procedurami (Action); jeśli są funkcjami (Func) wywołany zostanie tylko ostatni dodany do listy,
  • lista jest prywatna, dostęp do niej możliwy jest za pomocą metody GetInvocationList(),
  • lista jest edytowalna za pomocą przeciążonych operatorów += oraz -= (jak również metod Combine oraz Remove),
  • lista może zawierać duplikaty metod,

Na powyższym rysunku przedstawiono wewnętrzną budowę delegata

delegate string PrintVal( int val )

Jego lista inwokacyjna składa się z trzech metod:

  1. metody instancji string PrintX(int val) zdefiniowanej w klasie X
  2. metody statycznej string PrintY(int val) zdefiniowanej w klasie Y
  3. metody anonimowej

Przykład 1:

Podany poniżej kod odpowiada rysunkowi przedstawiającemu budowę delegata, z jedną zmianą: typ delegata opisuje procedurę (public delegate void PrintVal( int val );).

public delegate void PrintVal( int val );

public class X
{
  public int a;
  public X(int a) {this.a = a;}
  // zwraca wartość `a` poprzedzoną `n` kropkami
  public void PrintX(int n)  
    => Console.WriteLine(  (a.ToString()).PadLeft(n, '.') );
}

public static class Y
{
  // zwraca wartość `n` otoczoną nawiasami okrągłymi
  public static void PrintY(int n)  => Console.WriteLine( $"({n})" );
}

// Main
PrintVal print = null; // deklaracja zmiennej delegatowej
X x = new X(2);
print = x.PrintX; // faktyczne tworzenie isntancji delegata
Console.WriteLine( print.GetInvocationList() );
Console.WriteLine( print.GetInvocationList().Length );

// dodanie kolejnych metod do delegata
print += Y.PrintY;
print += delegate (int n){ Console.WriteLine( $"[{n}]" ); };
Console.WriteLine( print.GetInvocationList().Length ); //3
Console.WriteLine("-- Lista metod:");
foreach( var delegat in print.GetInvocationList() )
  Console.WriteLine( delegat.Method );

Console.WriteLine("-- Wywolanie MulticatsDelegate dla wartosci 5:");
if( print != null )
  print(5);

print?.Invoke(5); // alternatywne wywołanie z operatorem `null conditional`,
                  // równoważne poprzedniemu

Przykład 3:

Definicja typu delegatowego może uwzględniać parametry przekazywane przez referencję (ref, out).

Jeśli delegat ma parametry przekazywane przez referencję, wartość parametru może zmieniać się pomiędzy wywołaniami kolejnych metod z listy inwokacyjnej.

public delegate void ObliczenieDelegate(ref int x);

public class Liczydlo
{
  public void Inkrementuj(ref int x) { x++; }
  public void Dekrementuj(ref int x) { x--; }
  public void Podwojenie(ref int x) { x += x; }
}

// Main()
var licz = new Liczydlo();
ObliczenieDelegate f = null;
int x = 2;
f = licz.Inkrementuj;
f += licz.Podwojenie;
f += licz.Dekrementuj;
f += licz.Podwojenie;

Console.WriteLine($"x przed wywolaniem = {x}");
f(ref x);
Console.WriteLine($"x po wywolaniu = {x}");

Wynikiem kompilacji delegata jest klasa zapieczętowana z trzema metodami, których parametry i zwracane wartości bazują na deklaracji delegata.

// This is only pseudo-code!
public sealed class DelegateName : MulticastDelegate
{
  public delegateReturnValue Invoke(allDelegateInputRefAndOutParams);
  public IAsyncResult BeginInvoke(allDelegateInputRefAndOutParams, AsyncCallback cb, object state);
  public delegateReturnValue EndInvoke(allDelegateRefAndOutParams, IAsyncResult result);
}

Słowo kluczowe delegate pośrednio deklaruje klasę typu dziedziczącego z MulticasDelegate i Delegate. Twoje klasy nie mogą bezpośrednio dziedziczyć z tych klas bazowych (tylko kompilator to może zrobić).

Klasa Delegate jest traktowana przez Microsoft jako deprecated (do wycofania).
